Prerequisite
The course assumes that the students have:
  • Basic knowledge about Operating System and Compiler.
  • Familiarity with at least one programming language, such as C, C++, Java, or Python.
  • No prior security related research experience required.
Course Overview
From traditional PCs, mobile devices to IoT devices, software is literally ubiquitous in our everyday life. That is why the study of software security has become essential for defeating malware, perversing user privacy and protecting (intellectual and financial) properties. In this graduate-level research-oriented course, we will explore a variety of topics, including both classic and cutting-edge topics, in the area of software security. The goal of this course is to introduce basic concepts as well as state-of-the-art techniques in this area, and let students gain hands-on experience by applying some of these techniques to solve real-world security problems.

Lecture topics will include, but not limited to:
  • Traditional binary analysis, such as code searching, malware analysis
  • Mobile security, such as mobile app vulnerability detection, Android framework analysis
  • IoT security, such as policy enforcement, firmware analysis
  • Software testing, such as fuzzing, symbolic execution, hybrid analysis
  • Blockchain security, such as smart contract analysis
Course Outcomes
  • To learn basic concepts, threats, principles, and mechanisms in software security
  • To gain hands-on experience with state-of-the-art security analysis techniques
  • To develop the ability for analyzing and solving real-world security problems
Textbooks
There is no required textbook. Most readings will be from research papers in recent security conferences.

Course format and gradings
  • Paper presentation: 10%
  • Paper review: 10%
  • Lab assignments: 50%
  • Final Exam: 30%
  • Class participation (bonus): 5%



Lab Assignments
  • Lab1: Static analysis on Android applications
  • Lab2: Symbolic execution
  • Lab3: Blockchain smart contract analysis
Students can form groups (each group can have no more than 2 students) to work on lab assignments.
