Lab 3: Smart Contract Analysis

Handed out Nov 19 2022
Due Monday Dec 5 2022



1. Introduction

In this lab, you will implement a static analysis tool that performs static analysis on a smart contract and generates program dependence graphs (PDG) for each function within a given smart contract (provided by the instructor). Your tool will be based on an existing open-source smart contract static analyzer named Slither. Your tool takes as input source code of a smart contract, and outputs two things: 1) PDGs for each function in dot format; 2). statistics about these graphs.
You will be graded by reports. Randomly seletected students (~10%) are expected to explain your code and demonstrate that you understand what you did and why you did it that way.

2. Getting started

To get started:

a) Download the source code of the smart contract.

b) Install Slither and configure it up.



3. Assignment

a) Program Dependence Graph generation. Given a smart contract, your job is to output the PDG for each function within the smart contract into a dot format file. You can learn more about what PDG is and how to generate it from here.

b) statistics. You need to print out the following numbers in a file: 1). number of PDGs 2). number of nodes and edges within each PDG;



4. What to submit

You need to submit a report that includes the following:
  • Screenshot of your code (only your code)
  • Detailed explanation about the code
  • Screenshots about how you run the code and results
