Course Overview
Operating systems run the world, for without which, pretty much none of our modern lives would be the same. You wouldn't have web servers, desktop machines, laptops, phones, or smart watches; no Siri, no Alexa, no TensorFlow or CUDA programs.

In this course, we will get to know each major component of a computer system's softare stack. You will become familiar with both the high-level concepts underpinning modern operating systems and with low-level mechanisms, in addition to some new and emerging techniques and programming models for computer systems.

The success of this course heavily depends on hands-on lab assignments! We will be exploring a mini operating system named XV6, and implementing exciting new features on top of it! The assignments will bring together a lot of concepts you've learned in your previous courses, from data structures to computer architecture, virtual memory, and advanced C programming.

For this course, I assume that you have basic familiarity with computer organization (as introduced in CS 350), data structures, C, and systems programming (as covered in CS 351). You will for sure need to be able to program in C and should be able to navigate around a UNIX environment.

Lecture topics will include, but not limited to:
  • Process Scheduling/Synchronization
  • Memory Mamangement
  • File System
  • Computer Security

Course Outcomes
  • Define the functionality that modern operating systems must provide.
  • Articulate design tradeoffs inherent in operating system design.
  • Understand the benefits of software modularity and how it applies to OS design
  • Understand the importance of concurrency and how to implement concurrent abstractions correctly in an OS
  • Understand OS scheduling policies and mechanisms (for execution contexts, I/O)
  • Understand the benefits and applications of resource virtualization, and how it is achieved by the OS
  • Understand how operating systems implement protection, isolation, and access control
  • Understand the fundamentals of data storage, including file abstractions, file systems, object storage, etc.
  • Understand how OSes employ caching in various ways to improve performance, and how this impacts system complexity
  • Understand and apply programming patterns for efficient systems implementation
